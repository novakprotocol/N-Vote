# Teaching Guide

N-Vote separates public prioritization from private implementation.

## Public user path

1. Open N-Vote.
2. Submit a GitHub Issue using a public-safe form.
3. Vote using GitHub issue reactions.
4. Read Friday Top 5 and Approved Now markers on the public site.

## Owner path

1. Review public requests and votes.
2. Apply triage and owner approval labels.
3. Copy public-safe details into a private recon packet only after approval.
4. Keep private implementation details outside N-Vote.

## Safety rule

Public votes are not commands. They do not start a runner, mutate a private repository, or create private implementation work.
