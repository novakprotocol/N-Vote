# Public Labels

Create or maintain these labels in GitHub Issues:

- `area:n-suite`
- `area:overlay`
- `area:n-idea`
- `area:n-g-games`
- `area:n-g-chess`
- `area:n-g-pegsolitaire`
- `area:local-bridge`
- `area:docs`
- `type:feature`
- `type:bug`
- `type:question`
- `status:submitted`
- `status:triage`
- `status:needs-owner-approval`
- `status:approved-public`
- `status:private-packet-created`
- `approved-now`
- `friday-top5`
- `blocked`
- `do-not-implement-directly`

These labels are public markers only. They do not create private packets and do not start implementation work.
